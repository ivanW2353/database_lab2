# 日志说明

系统日志用于记录程序运行状态，主要用于课程展示和排查服务运行问题。用户借书、还书、预约等操作属于业务审计日志，单独写入 `logs/audit.log`。

## 1. 日志文件

系统运行日志文件位置：

```text
logs/runtime.log
```

日志文件会自动轮转，单个文件最大约 1 MB，最多保留 5 个历史文件。

业务审计日志文件位置：

```text
logs/audit.log
```

## 2. 查看方式

系统不在管理员界面展示日志。需要排查问题时，直接查看 `logs/runtime.log` 和 `logs/audit.log` 文件。

## 3. 记录内容

系统运行日志会记录以下事件：

- Flask 应用创建
- 数据库执行错误和后端请求异常

业务审计日志会记录以下事件：

- 登录成功、登录失败、退出登录
- 学生自助注册
- 学生修改密码、个人信息维护
- 学生立即借出图书
- 当前无可借馆藏时加入预约队列
- 学生选择未来日期提交预约
- 管理员新增图书、副本、附件
- 管理员新增或删除学生、管理员
- 管理员办理借书、还书
- 管理员处理预约状态
- 管理员登记违期罚金

## 4. 日志格式

日志格式示例：

```text
2026-06-02 10:30:00,000 INFO app_created host=127.0.0.1 port=8000
```

字段含义：

- 日志时间：事件发生时间
- 级别：`INFO`、`ERROR` 等
- 消息内容：运行事件、异常类型、数据库错误等
