# 管理员使用说明

本文档面向图书管理系统的管理员用户，说明管理员端的登录方式和主要管理功能。

## 1. 登录

打开系统首页后，选择管理员身份登录。

默认管理员账号：

```text
admin / 123456
```

## 2. 首页

管理员登录后进入首页，可以查看系统整体统计信息。

首页还会展示历史借阅排行前十的图书，便于管理员了解图书借阅情况。

## 3. 图书管理

管理员可以维护图书基础信息。

图书基础信息包括：

- ISBN
- 书名
- 分类
- 出版社
- 出版日期
- 版次
- 价格
- 语种
- 简介

表格支持点击表头进行升序或降序排序，默认按第一列升序排列。

## 4. 馆藏副本管理

管理员可以为已有图书新增馆藏副本。

副本使用条码区分，每一本实际可借的馆藏书都对应一个副本条码。借书和还书操作都以副本条码为依据。

## 5. 附件资料登记

管理员可以为图书登记图片、视频或文件资料。

系统通过 `book_media` 表保存文件路径和元数据，用于记录与图书相关的多媒体或附件资料。

## 6. 学生管理

管理员可以新增学生账号。

学生管理页面提供按学号和姓名查询学生的搜索框。

新增学生时，管理员只需要输入学号并点击“新增学生”。学生账号使用学号作为登录账号，学号需要符合中科大本科生学号格式要求。

新增成功后，系统会自动设置：

- 初始密码：`123456`
- 姓名：待完善
- 性别：其他
- 学生类型：本科生
- 学生状态：正常

姓名、学院、专业、电话、邮箱等个人信息由学生首次登录后自行填写。学生填写真实姓名后，姓名不再允许自行修改。

删除学生时，管理员输入学号并点击“删除学生”。如果该学生已有借阅或预约记录，系统不会直接删除，以避免破坏历史业务记录。

学生状态用于控制学生是否可以正常借阅，例如：

- 正常
- 暂停借阅
- 毕业

## 7. 管理员管理

管理员账号不开放公开注册，只能由已登录管理员新增。

管理员管理页面提供按工号和姓名查询管理员的搜索框。

新增管理员时需要填写：

- 工号
- 姓名
- 初始密码
- 电话
- 邮箱
- 职位

初始密码为空时默认为 `123456`。自定义初始密码至少 6 位，且只能包含数字或字母。

删除管理员时，输入工号并点击“删除管理员”。系统会进行二次确认；不能删除当前登录的管理员账号。如果目标管理员已有业务记录，系统会停用该账号而不是直接删除。

## 8. 办理借书

管理员在借书页面输入：

- 学号
- 图书副本条码

系统会根据学生状态、馆藏副本状态、借阅规则等信息办理借书。

借书成功后，系统会生成借阅记录，并将对应馆藏副本状态更新为借出。

## 9. 办理还书

管理员可以查看当前借出记录，并选择对应借阅记录办理还书。

还书成功后，系统会更新借阅记录和馆藏副本状态。如果存在超期情况，系统会生成或更新违期记录。

## 10. 预约处理

管理员可以查看学生提交的预约申请，并更新预约状态。

预约记录中会显示学生选择的计划借出日期。若学生是在无可借馆藏时通过“借出”加入队列，该字段可以为空。

预约状态显示为中文：

- 待处理
- 已通知
- 已完成
- 已取消
- 已过期

管理员可以根据图书到馆、学生是否取书等情况，将预约状态更新为已通知、已完成、已取消或已过期。

## 11. 违期与罚金

管理员可以查看学生违期记录，并登记罚金是否缴清。

违期记录包括：

- 学号
- 学生姓名
- 图书名称
- 违期天数
- 应缴罚金
- 已缴金额
- 缴清状态

## 12. 日志

管理员界面不提供日志查看功能。系统运行日志记录应用创建、请求异常、数据库执行错误等服务运行状态。

系统运行日志文件位置：

```text
logs/runtime.log
```

业务审计日志文件位置：

```text
logs/audit.log
```

日志详细说明见 `docs/日志说明.md`。
